This guide demonstrates how to create a custom storage implementation by extending `AbstractDocumentStorage`.

## What it demonstrates

- Creating a custom storage implementation
- Implementing the required persistence methods
- Supporting both encrypted and unencrypted documents with one class
- Building storage backends tailored to specific use cases

## Custom Storage Implementation

`AbstractDocumentStorage` implements the whole sync/update protocol for you. You only persist a `DocumentState` (the merged Y.js V2 update plus its encrypted content sidecars) and the document metadata. For unencrypted documents the sidecars are empty.

```typescript
import {
  AbstractDocumentStorage,
  type DocumentState,
  type DocumentMetadata,
} from "teleportal/storage";
import type { IndexedSidecar } from "teleportal/protocol/encryption";

export class MyCustomDocumentStorage extends AbstractDocumentStorage {
  async getDocumentState(key: string): Promise<DocumentState | null> {
    // Retrieve the stored update + sidecars from your backend
    return (await myBackend.getState(key)) ?? null;
  }

  async replaceDocumentState(
    key: string,
    update: Uint8Array,
    sidecars: IndexedSidecar[],
  ): Promise<void> {
    // Persist the already-merged update and its sidecars together
    await myBackend.storeState(key, { update, sidecars });
  }

  async writeDocumentMetadata(key: string, metadata: DocumentMetadata): Promise<void> {
    await myBackend.storeMetadata(key, metadata);
  }

  async getDocumentMetadata(key: string): Promise<DocumentMetadata> {
    return (await myBackend.getMetadata(key)) ?? defaultMetadata();
  }

  async deleteDocument(key: string): Promise<void> {
    await myBackend.deleteDocument(key);
  }
}
```
